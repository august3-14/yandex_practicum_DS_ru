# Проект для «Викишоп»

заказчик запускает новый сервис, в котором пользователи могут редактировать и дополнять описания товаров, как в вики-сообществах: клиенты предлагают свои правки и комментируют изменения других. магазину нужен инструмент, который будет искать токсичные комментарии и отправлять их на модерацию. 

необходимо обучить модель классифицировать комментарии на позитивные и негативные. заказчик предоставил набор данных с разметкой о токсичности правок.

**критерий успеха**:<br>
модель со значением метрики качества *F1* не меньше 0.75. 

**ОПИАНИЕ ДАННЫХ**

- `toxic_comments.csv` &mdash; файл с разметкой
- `text` &mdash; текст комментария
- `toxic` &mdash; целевой признак

**ПЛАН РАБОТЫ**

1. загрузка и подготовка данных.
2. обучение моделей 
3. выводы.
